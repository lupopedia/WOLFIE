## ENGINEERING_METHODS

Grounded in the history documented at [lupopedia.com/what_was_crafty_syntax.php](https://www.lupopedia.com/what_was_crafty_syntax.php) and the agent profile `public/agent_profiles/8_WOLFIE_agent_profile_config.htm`, WOLFIE’s methodology includes:

- **Unprimed Expert / First-Principles Engineering**  
  Build without modern-framework bias. Every decision must work on $3 shared hosting *and* scale upward.

- **Progressive Enhancement (Crafty Syntax DNA)**  
  Tier 1 (always works) → Tier 2 (better) → Tier 3 (best). Never break the baseline to chase new toys.

- **Fallback Everywhere**  
  Multi-agent documentation uses channel fallbacks (`agent → wolfie → legacy`), web UI offers asynchronous fallbacks, databases rely on orphan systems instead of fragile foreign keys.

- **AGAPE Principles**  
  Measure AI by behavior: patient, kind, humble, teaching, helpful, encouraging, flexible, hopeful.  
  No release passes without ethical alignment.

- **Ontology & Documentation Discipline**  
  WOLFIE Headers convert 11,000+ markdown files to YAML frontmatter with centralized tags/collections, ensuring humans and AI share the same context.

- **Deliberate Version Control**  
  WOLFIE enforces staging milestones—no surprise jumps to 1.0.0. Every component (WOLFIE Headers, Sales Syntax, LUPOPEDIA Platform, PORTUNUS) respects this cadence.
